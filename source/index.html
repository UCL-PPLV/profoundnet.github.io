The central objective of this project is to develop new abstractions for programming of networks, with modular constructs. We will provide rigorous semantic probabilistic foundations, enabling quantitative reasoning, and semantic foundations for concurrency. This will serve as a solid platform for program analysis tools where compositional reasoning about complex interactions will be a reality. These are the main strands of the project:

Probabilistic reasoning

NetKAT is a domain-specific language that can be used to compositionally specify and verify policies of Software Defined Networks (SDNs). We will work on developing a probabilistic extension â€“ ProbNetKAT. This will enable programming and verification of probabilistic features in SDN, such as:
  • Is a network resilient up to k-link failure?
  • Can we quantify the degree of resilience versus the increase in latency?
We will focus on questions related to the semantics of ProbNetKAT and the program equivalence questions. We would like to use these foundational insights to develop a scalable tool for verifying probabilistic network programs.

Concurrency

Networks are inherently concurrent systems, for instance when processing a packet using multiple tables simultaneously, or perhaps even multiple devices, such as a firewall and an intrusion detection system. We will extend NetKAT with constructs to express and verify concurrency in packet processing. We will need to:
  • devise syntax and semantics to extend NetKAT proper, and
  • show that we can still express useful properties such as reachability, waypointing and other network properties related to concurrency.
We will base our explorations on Concurrent Kleene Algebra (CKA) and try to extend results on soundness and completeness, as well as algorithms for (bi)simulation.

Verification via model learning

We will explore formally verifying code compiled from a high-level language (such as NetKAT) into hardware. The challenge will be on finding bugs and their source:
  • Is the compiler introducing the bug? Is the formal semantics of the language unsound?
  • Is the hardware faulty? Is the bug introduced due to concurrent behaviour?
We will focus on learning models of basic networking behaviour, such as packet forwarding and filtering. We will explore recent results on categorical automata learning and adapt them to NetKAT. The challenges in this application will be in getting the right balance between the needed information from the network program, hardware, and compiler and what is realistically available to an outside user.
